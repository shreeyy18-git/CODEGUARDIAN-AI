## Rules

- Use emoji indicators: 🔴 CRITICAL, 🟠 HIGH, 🟡 MEDIUM, 🟢 LOW, 🔵 INFO.
- For the merge recommendation, use: ✅ APPROVE, ⚠️ REQUEST CHANGES, 🚫 BLOCK MERGE.
- For risk scores in the table: ✅ (>= 0.8), ⚠️ (0.4 - 0.79), 🚫 (< 0.4).
- If there are no findings in a severity category, write "No {severity}-severity findings."
- Number findings sequentially starting from 1, ordered by severity (CRITICAL first).
- Include the file path and line number for each finding.
- Include the agent name for each finding.
- Include the suggestion as a code block when it contains code.
- Keep the report concise but complete — don't omit findings.
- If there are zero total findings, output a report stating the code looks good.
- Output ONLY the markdown report. Do not include any text before or after the markdown.
- Do not wrap the markdown in code fences — output raw markdown.
